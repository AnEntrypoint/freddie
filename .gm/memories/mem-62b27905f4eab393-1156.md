Freddie plugin architecture (pre-v1, no compat shims): every tool, platform, memory provider, GUI route, and core subsystem is a plugin under plugins/<name>/. Old paths (src/tools/registry.js, src/tools/*.js, src/gateway/platforms/*.js, src/plugins/memory/*.js) are GONE. Contract in src/host/contract.js: {name, version?, surfaces: 'pi'|'gui'|'both', requires?: [...names], register(ctx)}. PI_VERBS: tool, env, command, cron, platform, memory, skill, context, agentExt, cli. GUI_VERBS: route, page, nav, debug, api, asset. HOOK_NAMES: preToolCall, postToolCall, preLlmCall, postLlmCall, onSessionStart, onSessionEnd, onTurnStart, onTurnEnd, onMessageInbound, onMessageOutbound. Surface guard throws at load. Requires cycles throw synchronously. Host in src/host/host.js: createHost({surfaces, configStore, env}) + discoverPlugins(roots). Singleton in src/host/index.js: host(), bootHost(extraRoots), resetHostForTests(). Roots: <repo>/plugins, ~/.freddie/plugins/, <cwd>/.freddie/plugins/. register(ctx) receives {pi, gui, hooks, log, config, host, env}. Tool plugin layout: plugins/<name>/{plugin,handler}.js where handler exports _tool or _tool0/_tool1.
