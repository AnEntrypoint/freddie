Freddie multi-project workspace system: supports multiple isolated projects each with own home dir and plugin set. Registry at ~/.freddie/projects.json: {active, projects: [{name, path, created_at}]}. Default project (~/.freddie) protected from deletion. Code: src/projects.js (CRUD: loadRegistry, listProjects, getActiveProject, createProject, deleteProject, setActiveProject, applyActiveProjectFromRegistry), src/home.js (applyHomeOverride to set FREDDIE_HOME), src/host/index.js (bootHost calls applyActiveProjectFromRegistry before plugin discovery), plugins/gui-projects/plugin.js (GET/POST/DELETE /api/projects, POST /api/projects/active). Isolation: each project gets own sessions DB, config.json, skills/, plugins/, cron.db, batches/, logs/, auth.json under getFreddieHome(). Runtime switch CAVEAT: switching active project calls resetHostForTests() and clears caches but does NOT re-discover plugins in running dashboard — UI alerts user to restart. /api/health returns {freddie_home: "<active-project-path>"} after switch. Gap: in-process plugin re-discovery on project switch not implemented.
